# Deploying whereareyou-web

The app is live at **https://whereareyou.stu-bot.uk** (also reachable via
`way.stu-bot.uk`, which Cloudflare-redirects here). It's served by **GitHub
Pages from the `gh-pages` branch**, deployed by a manual force-push (there is no
CI workflow — the token lacks `workflow` scope).

## Live services it depends on

- **API:** `https://whereareyou-api.onrender.com` — Fastify + Redis on Render,
  `RESOLVER_MODE=apikey`, kept warm by a Cloudflare cron. Demo key is
  `demo-key-alpha`, baked into the build (not a secret — see `Resolve.tsx`).
- **Custom domain** `whereareyou.stu-bot.uk` is set via `public/CNAME`; DNS is
  Cloudflare grey-cloud → GitHub Pages. Don't change `public/CNAME`.

## The deploy procedure (exactly)

Three build-time env vars matter. **Miss `VITE_API_BASE` and session codes
break; miss `VITE_DEMO_API_KEY` and look-up 401s.**

```bash
cd whereareyou-web

# 1. Build for the live site (root base, wired to the deployed API + demo key)
VITE_API_BASE=https://whereareyou-api.onrender.com \
VITE_DEMO_API_KEY=demo-key-alpha \
  npm run build

# 2. SPA + Pages requirements
cp dist/index.html dist/404.html   # Pages has no SPA fallback; 404.html boots the app
touch dist/.nojekyll               # don't let Jekyll eat underscore paths

# 3. Force-push dist/ to the gh-pages branch (from a throwaway copy so the
#    working tree stays on main)
TMP=$(mktemp -d) && cp -r dist/. "$TMP/" && cd "$TMP"
git init -q && git checkout -q -b gh-pages && git add -A
git -c user.name="Stuart Haigh" -c user.email="stuart.haigh@opendialog.ai" \
  commit -q -m "Deploy"
git remote add origin https://github.com/stu-greenshoots/whereareyou-web.git
git push -q --force origin gh-pages
```

Then wait for the Pages build and verify (newly-added files can lag the CDN a
minute):

```bash
gh api repos/stu-greenshoots/whereareyou-web/pages/builds/latest --jq .status  # want "built"
curl -sL -o /dev/null -w "%{http_code}\n" https://whereareyou.stu-bot.uk/       # want 200
```

Also commit the source change to `main` normally (`git push origin main`) — the
gh-pages branch is build output only.

## Local dev

```bash
npm run dev        # https://localhost:5173 (self-signed cert; geolocation needs HTTPS)
```
Runs the API separately (`cd ../whereareyou-api && REDIS_URL=redis://localhost:6379 npm run dev`);
the dev server proxies `/v1` and `/health` to `localhost:8787`.

## Notes / gotchas

- **PWA service worker** is generated by `vite-plugin-pwa` (autoUpdate). After a
  deploy, an installed client updates on next load. `manifest: false` in the
  config — the hand-authored `public/manifest.webmanifest` is the one manifest.
- The build must serve from **root** on the custom domain, so leave `VITE_BASE`
  unset (defaults to `/`). It's only set to `/whereareyou-web/` for the bare
  `github.io` project-page path, which is not how the live site is served.
- The three-env-var build is easy to forget — that's the #1 way to ship a broken
  deploy here.
